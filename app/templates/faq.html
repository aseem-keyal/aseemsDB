User Questions

What makes this different from QuizDB?

My database does not display the questions inline with the webpage, instead you have to click the green links to open each packet. This is because displaying them inline requires parsing each question, while indexing them (knowing what text is in what file) is much easier to do. So in order to get to a question on this database, you have to search, open a file, and ctrl-f for what you want. It's kind of like Windows or Spotlight search but on the web and hopefully faster.


How many packets do you have in your database?

I should have everything that has been posted on quizbowlpackets, except vanity/trash sets that I feel don't have much study value. If I'm missing something you feel I should include, send me an email at the address on the about page.


What is all this weird lowercase text flooding my screen when I search?

That's how the backend software I use stores the packets. The reason that the search works well is it strips out all punctuation and case, and the snippet below each link is printed in this way. Some may find it useless, but quickly scanning it can sometimes be helpful in picking which links to open.


How do I search for only MS, HS, Collegiate questions?

Click on additional options and then change the folder. Since there are so many sets listed, an easy way is to press "M", "H", or "C" to just jump to the preferred difficulty. I've also classified everything before 2005 as "old", as it probably isn't as useful for studying as newer packets.


Why should people use this database over QuizDB?

This database isn't intended to replace QuizDB. While this database is more complete than QuizDB, looking at each question takes longer. If you want a couple questions on a given topic, QuizDB is a much better option. If you want to read every clue for a certain topic, then this database can do that for you. I personally love QuizDB for looking up something quickly and for its awesome randomization feature, something that my database is incapable of having.


What is the difference between all the search options?

The first option, Exact Search, looks for the exact phrase you search appearing anywhere in a packet. The second option, Answer Line, is used for search for tossups or bonuses on a certain subject (see finicky aspects of it below), and the third option searches for any packet containing all those words in any order and location. Option four is for if you are searching a single word and wish for packets containing all word forms of that word to appear in the results.


Why isn't the answer line search working?

The answer line search works by search "ANSWER: " then your query. Because of this, searching "Tolstoy" is unlikely to give a lot of results. A better idea is to search "Leo Tolstoy" or "* Tolstoy", as an asterisk works as a wildcard that can match any number of characters (Leo Tolstoy and Lev Nikolayevich Tolstoy).


Going to your database and typing in my query is really slow, do you have a better solution?

Yeah, if you're using Chrome then I'd highly recommend the Context Menu Search extension. This allows you to search my database whenever you highlight any text in your browser using your right click menu, if you're willing to do a little configuration.


This site kind of sucks on mobile. Are you going do anything to fix that?

Since my database works by linking you to pdf files, I can't think of any way to make it easier to use at the moment.


Technical Stuff

What software are you using to index the packets?

It's called recoll, and it's awesome. It can index almost anything you want, but I store these packets in pdf form on my server. This database was made when I found a webui on github for recoll and modified its appearance and functionality to fit the needs of searching quizbowl packets.


How long does it take to add a set to the database?

Around 5-10 minutes. I convert each file to a pdf, upload them to my server, and run a program that indexes the new files.
